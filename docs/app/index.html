Build Tool Guide

Each approach works differently. The right choice depends on what you are building, where it will run, and how much control you need.

Chat AI

Claude.ai, ChatGPT, Gemini

No setup, no code. You are not writing software. You are designing a system that runs in conversation with an AI tool. This might be a prompt library, a structured analytical framework, a repeatable decision-making workflow, or a document system. The product is a method, not an application.

Works with any AI subscription, including free tiers. No local setup required.

Examples

  • A competitive analysis framework you run once a month
  • A structured prompt system for reviewing content before publishing
  • A weekly planning workflow with built-in reflection questions

Vibe coding platform

Bolt.new, Lovable, Replit

Browser-based, no local setup. You build by describing what you want. The platform generates the code and runs it in the browser. No terminal, no file system. Most useful for form-based apps, dashboards, and simple data tools where the interface matters.

Works best when the data model is relatively simple and the app does not need deep control over back-end logic. The ceiling is lower than a custom-coded solution, but you can ship something working in hours.

Examples

  • A client-facing intake form that routes to a review dashboard
  • A simple inventory tracker with a clean interface
  • A shared team task board with status tracking

Coding agent

Claude Code, Cursor, Windsurf, Gemini Code Assist

Works in a local code editor. The agent reads and edits your actual files. This gives you complete control over the tech stack, data model, and build process. It is more technical than vibe coding, but an AI chatbot can walk you through the initial setup. No separate developer API subscription is required for most entry points.

The tradeoff: you need a local development environment (a terminal, installed software, and access to the file system). If you have never used one, there is a short learning curve upfront.

Examples

  • A financial planning tool with a local database and Monte Carlo simulation
  • A custom data pipeline that pulls from APIs and runs locally
  • A Python web application with a full back end and structured data model

No-code tools

Airtable, Notion, Zapier, Make, Glide

No code at all. These platforms give you a visual interface for building databases, automations, and simple apps. The ceiling is lower than a custom-coded solution, but setup is faster and maintenance is simpler. The right choice when your problem fits what these platforms already do well.

The main risk: discovering mid-build that your requirements exceed what the platform can handle. Be honest about that ceiling before you start.

Examples

  • A client project tracker in Airtable with automated status notifications
  • A Notion workspace with a structured intake form and review workflow
  • A Zapier automation that routes new leads to the right team member based on criteria

Hora's stage outputs work with all of these approaches. For coding agent projects, Stages 3 through 5 produce briefing documents you bring into Claude Code or Cursor. For all other approaches, they give you the context you need for your next AI conversation.

Getting Started

Hora guides you through building a purpose-built solution with AI, from the first question to a working result. It does not do the building for you. It helps you think clearly so the AI you are working with can.

The app is built around a five-stage framework: define the problem, scope the constraints, design the architecture, generate a build plan, then execute it. Each stage produces a document your AI tool can use. Learn more at the full framework.

Stage 1: Problem Definition. Articulate what is broken, why it matters, and what a good outcome looks like.

Stage 2: Constraint Scoping. Define hard limits, benchmark existing solutions, and prioritize features before you build.

Stage 3: Architecture. Design your system with AI as advisor. You make every decision — the AI explains the tradeoffs.

Stage 4: Build Plan. Generate a phased build plan, then review and confirm it before you start building.

Stage 5: Build. Execute your plan with AI guidance. Each session is briefed from your full project context.

Log a Decision. Capture any significant choice you make, at any stage. These accumulate in your Decision Journal and feed into later stage prompts.

Documents. Your auto-generated Project Context Document (updated after every session) and any reference files you upload. Copy or download them to paste into your AI tool.

Decision Journal. Your full decision history for this project, searchable and filterable. Edit or delete entries at any time.

Notes. Anything worth keeping that is not a formal decision: observations, ideas, open questions, follow-ups.

Log. A full chronological history of sessions, decisions, and plan changes.

Data and Privacy

All data is stored in your browser's local storage (IndexedDB). It is never sent to a server. Clearing your browser's cache or storage will delete it.

To protect your work, export a backup regularly from Settings. The backup is a single JSON file you can restore from the same Settings page. Your last backup date is shown there.

If you clear browser storage, your project data will be lost unless you have a backup. This is the main risk with a local-first app.

Hora will remind you to back up after three hours of active use. You can also back up manually at any time from Settings.

Go to Settings and click "Download backup (.json)." The file contains all your projects, stage records, decisions, sessions, and notes. Keep it somewhere safe.

To restore, go to Settings and click "Restore from backup." Select your backup file. This replaces all current data, so restore only from a backup you trust.

Installing Hora

Yes. Open Hora in your mobile browser, then use "Add to Home Screen" (iOS Safari) or "Install app" (Android Chrome) to add it as an app. It works fully offline once installed.

On iOS, export a backup before you install. iOS treats the installed app as a separate storage container from your browser tab. Any data you entered in Safari before installing will not carry over. Export a backup from Settings first, then restore it after installing. Android does not have this limitation.

On iOS: yes, unless you back up first. When you use "Add to Home Screen" on an iPhone or iPad, iOS creates a new isolated app container with its own storage. Data you entered in Safari before installing is not copied over. This is an Apple constraint that cannot be worked around.

Before installing on iOS, go to Settings and tap "Download backup (.json)." Once the app is installed and open, go to Settings and tap "Restore from backup" to bring your data across.

On Android and desktop (Chrome or Edge), installing Hora shares the same storage as your browser. Your data carries over automatically and there is nothing extra to do.

Installing Hora is closer to saving a bookmark than downloading software. The browser adds a shortcut to your desktop or home screen. No files are installed on your computer. If you change your mind, remove it the same way you would remove any app shortcut.

One note for iOS users: the installed app runs in its own storage container, separate from Safari. Back up your data before installing if you have existing projects. See "Will I lose my data if I install the app on my phone?" above.
